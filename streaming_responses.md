# Streaming Responses in Ollama JavaScript Library

The Ollama JavaScript library provides support for streaming responses, allowing you to process data in real-time as it's generated. This guide will explain how to enable streaming, process streamed data, and handle errors in streaming contexts.

## Enabling Streaming

To enable streaming for supported methods, set the `stream` option to `true` in the request object. The following methods support streaming:

- `chat`
- `generate`
- `create`
- `pull`
- `push`

Here's an example of enabling streaming for the `chat` method:

```javascript
import ollama from 'ollama'

const response = await ollama.chat({
  model: 'llama2',
  messages: [{ role: 'user', content: 'Tell me a story' }],
  stream: true
})
```

## Processing Streamed Data

When streaming is enabled, the method returns an `AsyncGenerator`. You can use a `for await...of` loop to process the streamed data:

```javascript
for await (const part of response) {
  console.log(part.message.content)
}
```

Each iteration of the loop will provide a part of the response. The structure of each part depends on the specific method being used.

## Handling Errors in Streaming Contexts

Errors can occur during streaming. To handle these errors, you can wrap your streaming code in a try-catch block:

```javascript
try {
  for await (const part of response) {
    console.log(part.message.content)
  }
} catch (error) {
  console.error('Streaming error:', error)
}
```

## Aborting Streamed Requests

The Ollama JavaScript library provides an `abort` method to stop all ongoing streamed requests:

```javascript
import ollama from 'ollama'

// Start a streamed request
const response = ollama.chat({
  model: 'llama2',
  messages: [{ role: 'user', content: 'Tell me a very long story' }],
  stream: true
})

// Process the stream
setTimeout(() => {
  ollama.abort() // This will abort all ongoing streamed requests
}, 5000)

try {
  for await (const part of response) {
    console.log(part.message.content)
  }
} catch (error) {
  if (error.name === 'AbortError') {
    console.log('Request was aborted')
  } else {
    console.error('Streaming error:', error)
  }
}
```

In this example, we start a streamed chat request and then abort it after 5 seconds. The `for await...of` loop will throw an `AbortError` when the request is aborted.

## Example: Streaming Chat Responses

Here's a complete example demonstrating how to use streaming with the `chat` method:

```javascript
import ollama from 'ollama'

async function streamingChatExample() {
  const response = await ollama.chat({
    model: 'llama2',
    messages: [{ role: 'user', content: 'Write a short story about a robot learning to paint' }],
    stream: true
  })

  try {
    for await (const part of response) {
      process.stdout.write(part.message.content)
    }
  } catch (error) {
    console.error('Streaming error:', error)
  }
}

streamingChatExample()
```

This example will print the story to the console in real-time as it's generated by the model.

## Conclusion

Streaming responses in the Ollama JavaScript library allow for real-time processing of generated content. By enabling the `stream` option and using `AsyncGenerator`s, you can create more responsive and efficient applications that can handle large amounts of generated text or process long-running tasks.
